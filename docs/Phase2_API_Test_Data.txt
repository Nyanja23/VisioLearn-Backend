# API Test Data: Phase 2 Authentication & User Endpoints

Follow these steps exactly in your Swagger UI (http://localhost:8000/docs) to verify that the RBAC, user creation, and JWT logic are functioning properly.

---

### Step 1: Create an Admin User
**Endpoint:** `POST /api/v1/users/`
**Purpose:** Creates our first user with administrative privileges to act as a baseline.

| Parameter | Value |
| :--- | :--- |
| `email` | `admin@visiolearn.org` |
| `password` | `SecurePass123!` |
| `full_name` | `System Administrator` |
| `role` | `admin` |
| `school_id` | `null` |

**Expected Output (HTTP 201 Created):**
```json
{
  "email": "admin@visiolearn.org",
  "full_name": "System Administrator",
  "role": "admin",
  "school_id": null,
  "id": "e4f8b9... (UUID)",
  "created_at": "2026-03-27T...",
  "is_deleted": false
}
```

---

### Step 2: Test Login (Generate JWT and Refresh Token)
**Endpoint:** `POST /api/v1/auth/login`
**Purpose:** Verifies password hashing and issues our short-lived access token and 90-day refresh token.

| Field | Value |
| :--- | :--- |
| `email` | `admin@visiolearn.org` |
| `password` | `SecurePass123!` |

**Expected Output (HTTP 200 OK):**
```json
{
  "access_token": "eyJhbG... (Long JWT String)",
  "refresh_token": "eyJhbG... (Another Long JWT String)",
  "token_type": "bearer"
}
```
*(ACTION: Copy the `refresh_token` string to your clipboard for Step 3. Click the "Authorize" button at the top of swagger and paste your `access_token` so you can test secure endpoints later).*

---

### Step 3: Test Refresh Token (Session Rotation)
**Endpoint:** `POST /api/v1/auth/refresh`
**Purpose:** Validates the refresh token logic designed for intermittent offline connections, revokes the old token, and issues a new pair.

| Base JSON Key | Value |
| :--- | :--- |
| `refresh_token` | *(Paste the exact refresh_token you copied from Step 2)* |

**Expected Output (HTTP 200 OK):**
```json
{
  "access_token": "eyJhbG... (New Long JWT)",
  "refresh_token": "eyJhbG... (New Long JWT)",
  "token_type": "bearer"
}
```
*(Note: If you attempt to send the exact same payload again, it should fail with a `401 Unauthorized "Token has been revoked or safely deleted"` error, proving the token rotation works!)*
